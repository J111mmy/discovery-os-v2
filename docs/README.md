# DiscOS docs

Reference docs, briefs, and assessments. **Live anchors stay at the repo root**
(`CLAUDE.md`, `AGENTS.md`, `README.md`, `ROADMAP.md`, `OPUS_CODEX_CHANNEL.md`).

| Folder | What's in it |
|--------|--------------|
| `briefs/codex/` | `CODEX_BRIEF_*` — implementation briefs for Codex. Each carries the security-gate banner; the gate itself lives in root `AGENTS.md`. |
| `briefs/design/` | Design briefs incl. `DESIGN_BRIEF_APP_REBRAND_IMPLEMENTATION.md` (prototype → live app) and the existing UX-refresh / AI-assisted-edit briefs. Prototype source: `New DiscOS app design/` at repo root. |
| `security/` | Security assessments, hardening notes, gate reviews. **Start at `SECURITY_POSTURE.md` — the living invariants ledger to re-check as we build.** Then the dated assessments (`PROMPT_INJECTION_ASSESSMENT_2026-06-06`, `ARCHITECTURE_SECURITY_HARDENING`, `SECURITY_ASSESSMENT_MILESTONE_0`, `GATE2/GATE3`, assessor notes). |
| `product/` | Product/PRD/positioning/monetization (`PRD_COMPETITIVE_INTEL`, `BRAND_POSITIONING_BRIEF`, `NAMING_EXPLORATION`, `PHASE-2-BRIEF`, billing/monetization, `MILESTONE_0_CHECKLIST`). |
| `process/` | Working process docs (`CODEX_TASKS`, `CLAUDE_CODE_HANDOFF`). |
| `ops/` | Deploy/runbook (`DEPLOY`). |
